# App Store Connect 제출 체크리스트 (#4 메타데이터)

코드로 해결되지 않는 **제출 단계 필수 입력** 항목입니다. 아래 값 그대로 입력하면 됩니다.

## 1. 개인정보 처리방침 URL (필수)

사진 라이브러리 접근 앱은 개인정보 처리방침 URL이 **반드시** 필요합니다. 이 앱은 데이터를
개발자 서버로 수집/전송하지 않으므로 아래 1문단짜리 페이지로 충분합니다(GitHub Pages, Notion 공개 페이지 등):

> PhotoMove Bridge는 사용자가 선택한 사진/동영상을 앱 내부에서 USB 내보내기 파일로 만듭니다.
> 사용자는 이 파일을 USB 파일 공유로 자신의 Windows PC에 직접 복사할 수 있습니다.
> 앱은 사진/동영상을 개발자 서버, 클라우드, 네트워크로 전송하지 않으며,
> 분석·광고 SDK를 사용하지 않습니다. 사진 접근 권한은 사용자가 선택한 항목을
> USB 내보내기 폴더로 복사하고, 사용자가 직접 확인한 항목을 삭제하는 데에만 사용됩니다.
> 문의: <연락 이메일>

## 2. App 개인정보(Privacy Nutrition Label)

App Store Connect → **앱 개인정보 보호** → 설문:

- **데이터를 수집합니까?** → **아니요 (No, we do not collect data)**
  - 사진은 개발자/제3자 서버로 전송되지 않고 사용자가 자신의 PC로 직접 내보내는 로컬 파일이므로 "수집(collect)"에 해당하지 않습니다.
  - 제3자 SDK/분석/광고 없음.

## 3. 수출 규정(Export Compliance)

- Info.plist에 `ITSAppUsesNonExemptEncryption = false` 가 설정되어 있으므로, 업로드 시
  암호화 질문이 자동으로 처리됩니다.
- 빌드별로 묻는다면: "암호화 사용 = 예" → "면제 대상(표준 암호화/해시만 사용)" → SHA-256 해시.

## 4. 연령 등급(Age Rating)

- 전 연령(4+). 부적절 콘텐츠 없음. 사용자 생성 콘텐츠는 본인 사진 로컬 처리만.

## 5. 카테고리 / 메타데이터

- 1차 카테고리: **유틸리티(Utilities)** (또는 사진 및 비디오)
- 스크린샷: 6.7"/6.5" iPhone 필수. 권한/사진선택/이동/결과 화면 캡처 권장.
- 지원 URL 필요(간단한 안내 페이지 가능).

## 6. 심사 정보 메모

- `docs/APP_REVIEW_NOTES.md` 의 영문 노트를 **App 심사 정보 > 메모**에 붙여넣기.
- 데모 계정: 불필요(로그인 없음). 체크 해제.

## 7. 제출 전 마지막 점검

- [ ] 개인정보 처리방침 URL 입력
- [ ] 앱 개인정보: "데이터 수집 안 함" 선택
- [ ] 심사 메모에 영문 노트 붙여넣기
- [ ] 스크린샷/아이콘 업로드
- [ ] (권장) 라운드트립 데모 영상 첨부
- [ ] 실기기 + Windows PC + 외장하드 대용량 세션 가져오기 1회 테스트
